Detailed Description

Context class.

The context class exposes parts of the ZeroMQ C API in an object-oriented manner. Only the functionality we require is implemented. The context object implicitly behaves like a pointer to void which is what ZeroMQ expects. Thus a context object can be used like a regular context pointer with the C API. For example the zmq_ctx_set is not implemented but can be used nevertheless.

if ( zmq_ctx_set(context, ZMQ_IPV6, 1) == -1 )
throw zmq::error();

When using the C API directly, errors have to be checked explicitly!

For details see http://api.zeromq.org/.
