1 # Eigen with CUDA
2 
3 ## Problem
4 
5 A small subset of Eigen can [run in CUDA
6 kernels](http://eigen.tuxfamily.org/dox/TopicCUDA.html). This is
7 mainly limited to fixed size matrices but one can already do a lot
8 with them. What one can *not* do is compute the inverse, i.e. this
9 kernel will not compile:
10 ```cpp
11 template < typename MatrixType, typename ResultType >
12 __global__ void inverse_kernel(MatrixType * matrix, ResultType * inverse)
13 {
14  *inverse = matrix->inverse();
15 }
16 ```
17 with an error similar to this
18 ```
19 error: calling a __host__ function(...) from a __global__ function(...) is not allowed
20 ```
21 
22 ## Solution
23 
24 Functions in the Eigen core are not sufficiently marked as
25 `__device__`. Digging through the source code one can find that the
26 internal `compute_inverse` functor is sufficiently marked (which is
27 what computes the inverse under the hood). This does not perform any
28 check whether the matrix is invertible! It also might not work for
29 dynamically sized matrices.
